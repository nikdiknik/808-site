# Telegram-бот для подбора методик для музыкантов

Бот собирает у пользователя:
- уровень опыта в музыке;
- этап, на котором произошёл ступор;
- дополнительное текстовое описание, если выбран вариант `Другое`.

После этого бот отправляет данные в OpenAI вместе с TSV-таблицей с методиками и возвращает пользователю:
- наиболее подходящую методику;
- краткое описание этой методики;
- объяснение, почему она подходит;
- практические шаги;
- дополнительные советы.

## Что нужно для запуска

1. Python 3.11+.
2. Токен Telegram-бота от BotFather.
3. API-ключ OpenAI.

## Установка

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

## Настройка переменных окружения

Скопируй `.env.example` в `.env` и заполни значения:

- `TELEGRAM_BOT_TOKEN` - токен Telegram-бота.
- `OPENAI_API_KEY` - ключ OpenAI.
- `OPENAI_MODEL` - модель OpenAI, по умолчанию `gpt-4.1-mini`.
- `TSV_PATH` - путь к TSV-файлу с методиками. Можно относительный, от папки проекта.
- `CHECKLIST_PATH` - путь к изображению чеклиста. Можно относительный, от папки проекта.
- `ANALYTICS_PATH` - путь к JSON-файлу, где хранится аналитика по пользователям и запускам.

Если не хочешь использовать `.env`, можно просто задать эти переменные в системе вручную.

## Запуск

```bash
python bot.py
```

## Логика сценария

1. Пользователь запускает бота командой `/start`.
2. Бот показывает приветствие и объясняет механику.
3. Пользователь выбирает уровень опыта.
4. Пользователь выбирает тип ступора.
5. Если выбран вариант `Другое`, бот просит ввести текстом проблему.
6. Бот отправляет данные в OpenAI и возвращает ответ.
7. Бот показывает ответ по шагам с кнопкой `Далее`.
8. После прохождения сценария бот может отправить чеклист по кнопке `Получить чеклист`.
9. После первого полного прохождения для повторного запуска нужна подписка на канал `https://t.me/demos_808`.
10. Бот сохраняет аналитику по уникальным пользователям, количеству запусков и завершений сценария.
11. По команде `/stats` бот показывает сводку по использованию.

## Важно

- Бот читает TSV-файл локально и передаёт его содержимое в OpenAI как текст.
- `TSV_PATH`, `CHECKLIST_PATH` и `ANALYTICS_PATH` можно задавать относительными путями от папки проекта.
- Для кнопки `Получить чеклист` в папке проекта должен лежать файл `Чеклист_808.png`, если не задан другой `CHECKLIST_PATH`.
- Для проверки подписки бот должен быть добавлен в канал `@demos_808`, лучше как администратор.
- Аналитика хранится в локальном JSON-файле `analytics.json`, если не задан другой путь через `ANALYTICS_PATH`.
- По умолчанию бот работает через polling, без вебхуков.
